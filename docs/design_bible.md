## РАЗДЕЛ 1. КАРТА НАВЫКОВ ДИЗАЙНЕРА (SKILL CARDS)

### Карта навыка: Visual Hierarchy (Визуальная иерархия)
```yaml
id: "DB-SKILL-001"
title: "Visual Hierarchy"
category: "Visual Design"
difficulty: "Expert"
```
*   **Описание**: Управление вниманием пользователя на двухмерной плоскости экрана посредством распределения визуального веса элементов. Навык опирается на физиологию человеческого зрения (саккады, фовеальное и периферическое зрение) и законы гештальт-психологии.
*   **Цель**: Провести взгляд пользователя по заранее спроектированному визуальному пути (Visual Path) для последовательного считывания информации и бесконфликтного достижения целевого действия (CTA).
*   **Когда используется**: Проектирование структуры первого экрана (Hero Section), верстка сложных информационных дашбордов, структурирование прайс-листов и контентных страниц.
*   **Влияние на качество дизайна**: При высоком уровне навыка время считывания ценностного предложения (Value Prop) составляет до 2.5 секунд; показатель отказов (Bounce Rate) снижается. При низком уровне возникает визуальный шум, когда элементы спорят друг за друга за внимание.
*   **Связанные навыки**: Typography Scale (Типографическая шкала), Composition & Balance (Композиция и баланс), Contrast Control (Контроль контрастности).
*   **Чек-лист оценки**:
    - [ ] Выделен один доминантный элемент на первом экране, занимающий максимальный визуальный вес.
    - [ ] Субдоминантные элементы отстают от доминанты минимум на 30% по визуальному весу (размер, контраст).
    - [ ] Расстояния между логическими блоками больше, чем расстояния между элементами внутри блоков (Закон близости).
    - [ ] Проведен Blur-тест: при размытии макета на 20px ключевые акценты остаются различимыми в правильном порядке.
*   **Примеры правильного применения**: Stripe — крупный контрастный заголовок H1, яркий вторичный градиентный фон-акцент, кнопка CTA имеет максимальный контраст, второстепенные ссылки визуально приглушены.

### Карта навыка: Composition & Visual Weight (Композиция и визуальный вес)
```yaml
id: "DB-VIS-002"
title: "Composition & Visual Weight"
category: "Visual Design"
difficulty: "Advanced"
```
*   **Теория визуального веса**: Визуальный вес — это мера того, насколько сильно объект привлекает внимание зрителя. Факторы, определяющие визуальный вес:
    1.  *Размер*: Крупные объекты кажутся тяжелее.
    2.  *Цвет и насыщенность*: Высококонтрастные, насыщенные и теплые цвета (красный, оранжевый) физиологически воспринимаются как более тяжелые, чем холодные и приглушенные (синий, серый).
    3.  *Форма*: Сложные, нерегулярные геометрические формы обладают большим визуальным весом, чем простые круги или квадраты аналогичной площади.
    4.  *Текстура и глубина*: Объекты с выраженной текстурой, объемом (3D) или тенями выглядят тяжелее плоских элементов.
*   **Математические правила композиционного баланса**: Баланс бывает симметричным (формальным) и асимметричным (динамичным). При асимметричном балансе точка равновесия рассчитывается по принципу рычага:
    $$Визуальный \, Вес_1 \times Дистанция_1 = Визуальный \, Вес_2 \times Дистанция_2$$
    Маленький тяжелый элемент (например, яркая контрастная кнопка) на краю экрана может сбалансировать массивный, но блеклый блок текста, расположенный ближе к центру.
*   **Практические рекомендации**:
    *   *Правило 60-30-10*: 60% площади отдается под доминирующее визуальное пространство (воздух и фоновый цвет), 30% — под поддерживающую структуру (контентные карточки, текст), 10% — под акцентные пятна (CTA, интерактивные элементы).
    *   *Использование силовых точек*: При верстке по асимметричной сетке размещайте ключевые смысловые узлы на пересечениях третей (Rule of Thirds).
*   **Чек-лист проверки**:
    - [ ] Проверен баланс левой и правой половин макета. Нет ощущения "заваливания" одной из сторон.
    - [ ] Реализован принцип "внутреннего и внешнего": внутренние отступы элементов меньше внешних отступов контейнеров минимум в 1.5 раза.

## РАЗДЕЛ 2. UI-ПАТТЕРНЫ (UI PATTERNS DIRECTORY)

### Паттерн: Hero Section (Split-Screen Content + Visual)
```yaml
id: "DB-PAT-001"
title: "Hero Section: Split-Screen Content + Visual"
category: "UI Patterns"
conversion_rate: "High"
```
*   **Описание**: Разделение первого экрана на две основные колонки в соотношении 50/50 или 60/40. В левой колонке располагается текстовый блок (H1, дескриптор, CTA, Social Proof), в правой — медиаконтент (3D, статичный рендер, интерактивный интерфейс или видео).
*   **Бизнес-задачи**:
    *   Быстрое вовлечение пользователя через визуальную демонстрацию продукта.
    *   Снижение когнитивной нагрузки за счет четкого разделения зон чтения и визуального восприятия.
*   **Когда применять**: SaaS-продукты со сложным интерфейсом; физические премиум-товары (справа высококачественный 3D-рендер или фото).
*   **Конверсионные метрики**: По данным A/B тестов, сплит-экран с интерактивным UI-скриншотом справа повышает конверсию в регистрацию на SaaS-платформах на **18.4%** по сравнению с абстрактными иллюстрациями.

## РАЗДЕЛ 3. UX & ИНТЕРАКЦИОННЫЙ ДИЗАЙН

### Спецификация: Информационная архитектура (Information Architecture)
```yaml
id: "DB-UX-003"
title: "Information Architecture: Hierarchy & Navigation"
category: "UX"
difficulty: "Expert"
```
*   **Построение ментальных моделей**: Информационная архитектура должна соответствовать ожиданиям пользователя, основанным на его прошлом опыте (Закон Якоба).
    *   *Плоская архитектура (Flat IA)*: Минимальное количество уровней вложенности (не более 3). Идеально для промо-сайтов, небольших e-commerce и SaaS.
    *   *Глубокая архитектура (Deep IA)*: Сложная структура (4 и более уровней). Требует применения фасетной фильтрации, хлебных крошек (Breadcrumbs) и мега-меню.
*   **Чек-лист оценки UX навигации**:
    - [ ] Время перехода к любому целевому конверсионному экрану не превышает 3 кликов.
    - [ ] Нажатие на логотип всегда возвращает на главный экран.
    - [ ] Активный пункт меню визуально выделен с контрастностью не менее 4.5:1 по стандартам WCAG AA.

## РАЗДЕЛ 4. ПСИХОЛОГИЯ ВОСПРИЯТИЯ (LAWS OF UX)

### Закон Фиттса (Fitts's Law)
```yaml
id: "DB-PSY-001"
title: "Fitts's Law"
category: "Psychology"
difficulty: "Intermediate"
```
*   **Математическое определение**: Время достижения цели зависит от расстояния до нее и ее размера. Формула Шеннона (модифицированная версия закона Фиттса):
    $$MT = a + b \log_2 \left( \frac{2D}{W} \right)$$
    Где $MT$ — время движения, $D$ — дистанция до цели, $W$ — ширина цели (размер кнопки или зоны клика).
*   **Практическое применение**:
    *   *Увеличение целевой зоны*: Минимальный размер тач-таргета составляет **48x48px** (Apple HIG) или **48x48dp** (Google Material Design).
    *   *Размещение на границах и углах*: Кнопки на краях экрана легче нажимать, так как указатель мыши «упирается» в край экрана, делая ширину цели фактически бесконечной.
    *   *Минимизация дистанции*: Всплывающие контекстные меню должны появляться непосредственно рядом с курсором пользователя.

---

### Закон Хика (Hick's Law)
```yaml
id: "DB-PSY-002"
title: "Hick's Law"
category: "Psychology"
difficulty: "Intermediate"
```
*   **Теория и математическая модель**: Время, необходимое для принятия решения, увеличивается логарифмически с ростом числа и сложности альтернатив.
    $$T = b \cdot \log_2(n + 1)$$
    Где $T$ — время принятия решения, $n$ — количество альтернатив.
*   **Практическое применение**:
    *   *Снижение когнитивной нагрузки*: Ограничивайте количество вариантов выбора на критических этаках. В тарифной сетке оптимально показывать не более 3–4 планов.
    *   *Постепенное раскрытие (Progressive Disclosure)*: Сложные многошаговые формы разделяются на логические шаги вместо вывода всех полей на одном экране.

## РАЗДЕЛ 8. DESIGN SYSTEMS & COMPONENT SPECS

### Компонент: Primary Button (Кнопка основного действия)
```yaml
id: "DB-DS-001"
title: "Component Specification: Button"
category: "Design Systems"
```
*   **Архитектура состояний (States Architecture)**: Каждая кнопка должна иметь 6 состояний: `[ Default ] -> [ Hover ] -> [ Focus ] -> [ Active ] -> [ Loading ] -> [ Disabled ]`.
*   **Стилизация (CSS)**:
```css
.btn-primary {
  display: inline-flex;
  align-items: center;
  justify-content: center;
  font-weight: 500;
  transition: all 0.2s cubic-bezier(0.4, 0, 0.2, 1);
  background-color: var(--color-primary-600);
  color: var(--color-white);
  border: 1px solid transparent;
}
.btn-primary:hover {
  background-color: var(--color-primary-700);
  cursor: pointer;
}
.btn-primary:focus-visible {
  outline: 2px solid var(--color-white);
  outline-offset: -2px;
  box-shadow: 0 0 0 4px var(--color-primary-300);
}
```
*   **Доступность (Accessibility / WCAG 2.1)**:
    *   *Контрастность*: Коэффициент контрастности текста кнопки к цвету фона кнопки должен быть $\ge 4.5:1$.
    *   *Клавиатурная навигация*: Переход по Tab должен вызывать состояние `:focus-visible`. Активация — по Space и Enter.

## РАЗДЕЛ 11. МАТРИЦА ОЦЕНКИ ДИЗАЙНА (HEURISTICS EVALUATION SHEETS)

Каждый макет оценивается по 5-балльной шкале по следующим критериям:

| Критерий | Вес | Формула валидации | Критический показатель |
| :--- | :--- | :--- | :--- |
| **Contrast (WCAG AA)** | 0.25 | $C = (L_1 + 0.05) / (L_2 + 0.05)$ | Должен быть $\ge 4.5:1$ для текста |
| **Typographic Rhythm** | 0.20 | $H_{line} = LineHeight \times FontSize$ | Должен быть кратен 4px или 8px |
| **Focal Points** | 0.20 | Blur Test 20px (Count of distinct nodes) | Не более 2 доминирующих точек на экран |
| **Interaction Target** | 0.20 | Size of Interactive Areas | $\ge 48\text{px}$ на мобильных устройствах |
| **Cognitive Load** | 0.15 | Option Selection Count (Hick's Law) | Меньше 5 вариантов на этапе CTA |
