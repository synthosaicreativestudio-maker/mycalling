# 27 — План доработки глубокой коуч-сессии (методология + баги)

**Дата:** 2026-07-20
**Статус:** план (решения владельца зафиксированы)
**Связано с:** docs/26 (Этап 5–7), методология из чата.

## Context

Тестирование глубокой сессии вскрыло три проблемы:
1. **Пустая «Карта талантов».** Ученик прямо сказал «хочу свою IT-компанию»,
   «своя компания, влияние, слава, деньги» — а колесо: tech = 0 %, startup = 0 %,
   только creative 30 % / организация 20 %. Deep-сессия «слушала, но профиль пуст».
2. **Ярлык шага рассинхронён.** Шапка показывала «Шаг 8 из 22: Страхи и барьеры»,
   пока коуч уже вёл работу с целью (пирамида), затем скакнула на «Шаг 16:
   Глубинная цель». Виден переход «страхи → цель».
3. **Методологический разрыв.** В глубоком потоке нет отдельного этапа сбора
   талантов (они лишь домысливаются моделью) и нет этапа «Барьеры/убеждения»
   (WOOP Obstacle) — страхи в deep теряются.

### Первопричина
- «Карта талантов» собирается **экспресс-портретом** (хобби, предметы, опыт, тип
  мышления) + **тестами** (RIASEC/BFI/ICAR/VIA/PVQ). Глубокая сессия (пирамида
  Дилтса) работает с **целью и смыслом**, а не с талантами.
- Ученик зашёл в глубокую **до экспресса** (deep-first) → интересы никто не
  спросил → колесо пусто. Плюс экстрактор `talentScores` недооценивает явные
  сигналы («IT-компания» → tech = 0), а слияние идёт по `Math.max` (только вверх),
  поэтому 0 так и остаётся 0.

### Решения владельца
- **B: в глубокую сессию можно только после экспресс-портрета.** Сначала узнаём
  человека (таланты собраны), потом углубляемся.
- **Добавить этап «Барьеры и убеждения» (WOOP Obstacle)** между «Идентичность» и
  «План».

## Целевая методология глубокой сессии (после экспресса)
Пирамида Дилтса + WOOP + CLEAR + «Внутренняя игра». 7 рабочих этапов + финал:
1. Цель (Хочу) · 2. Образ результата · 3. Эмоции · 4. Идентичность ·
5. **Барьеры и убеждения (НОВЫЙ)** · 6. План/способности · 7. Первый шаг · Финал.
Сквозняк: на этапах 1–2 коуч дособирает сигналы интересов (страховка для колеса).

## Трек 1 — Гейт «только после экспресса» (решение B)

**Суть:** нельзя стартовать глубокую, пока не собран экспресс-портрет
(`allPsychologyCollected`). Убирает и deep-first, и блуждание счётчика по
экспресс-шагам.

**Правки:**
- `app/coach/page.tsx`: `handleSelectMode('DEEP')` (~452) и forceDeep-путь (~278) —
  не запускать DEEP, если экспресс не завершён; вместо этого вести/дозавершить
  экспресс, а кнопку/карточку выбора DEEP показывать только при `coachCompleted`.
- `app/api/v1/coach/chat/route.ts`: при запросе `sessionMode: 'DEEP'` с несобранным
  экспрессом — сервер остаётся в экспресс-сборе (уже так на строках 440–468), но
  надо **не помечать сессию DEEP до готовности** (иначе ярлыки/пирамида включаются
  рано). Переводить в DEEP-режим только когда `allPsychologyCollected`.
- `app/assessment/page.tsx`: предложение «пройти глубокую» уже гейтится
  `coachSessionMode==='EXPRESS'` — оставить как есть (это корректный вход).

**Критерий:** зайти в глубокую до экспресса невозможно; после экспресса —
пирамида начинается сразу с «Цель», без экспресс-ярлыков.

## Трек 2 — Новый этап «Барьеры и убеждения» (WOOP Obstacle)

**Правки (аккуратно — 63 места со ссылками на 16–22 в route):**

- `app/api/v1/coach/chat/route.ts`: новое поле `deepBarriers` (checks `hasDeepBarriers`,

- Клиент: `app/coach/page.tsx` STEP_TITLES (+20 «Барьеры и убеждения», сдвиг),

- Отчёт: показать «Барьеры и убеждения» в coachSection и в печатной версии.

**Критерий:** глубокая сессия проходит 7 этапов + финал; страхи/убеждения
собираются и попадают в отчёт.

## Трек 3 — Починка ярлыка шага в DEEP

**Суть:** в DEEP-режиме шапка должна показывать **пирамидный** ярлык
(«Этап X из 8: …»), а не экспресс-ярлык («Шаг 8: Страхи и барьеры»).

**Правки:** `app/coach/page.tsx` — логика заголовка (строки 27–49, 717): выбирать
набор ярлыков по `isDeepMode`, а не по абсолютному номеру шага. С Треком 1 (B)
deep всегда стартует ≥16, поэтому пересечения с экспресс-ярлыками не будет.

**Критерий:** в глубокой сессии в шапке только пирамидные названия, без «Страхи»
во время работы с целью.

## Трек 4 — Усиление сбора интересов (колесо не должно быть пустым)

**Суть:** экстрактор `talentScores` обязан зажигать явные сигналы.

**Правки:** `app/api/v1/coach/chat/route.ts` (инструкция talentScores, ~885):
- добавить явные маппинги и few-shot: «IT-компания / программирование / робот /
  сайт → **tech**»; «свой бизнес / стартап / влияние / лидер / масштаб → **startup**»;
  «дизайн / тексты / музыка / рисование → **creative**»; «исследование / формулы /
  анализ → **analytical**»; «помогать / учить / общаться / продавать → **social**»;
  «порядок / процессы / организовывать → **organizational**»;
- правило: если в реплике есть явный сигнал сферы — ставить ≥ 40 по шкале, НЕ 0;
  «строго 0» — только при полном отсутствии сигнала;
- собирать интересы уже на этапах 1–2 глубокой (цель/результат), а не только в
  экспрессе.

**Критерий:** для реплики «хочу свою IT-компанию» tech и startup > 0 в колесе.

## Трек 5 — Дизайн колеса и пирамиды + светлая тема

**Суть:** на скринах колесо/пирамида ломаются на светлой теме и переполняются.

**Правки:**
- **Жёлтый таб:** `app/coach/page.tsx` (~1020) — активная кнопка «Пирамида целей»
  `bg-[#EAB308] text-white` захардкожена и нечитаема на свету. Заменить на
  тему-токен (`var(--warning)` — уже капучино-осознанный, или `var(--accent-brown)`
  для консистентности с «Колесом») + читаемый текст.
- **Текст на свету:** `app/coach/PyramidOfAlignment.tsx` — весь текст
  `text-white`/`text-slate-400/300/500` невидим на светлой панели (`bg-white/15`).
  Перевести на тему-осознанные токены (`text-[var(--text)]`/`[var(--text-muted)]`
  или `dark:` варианты). Аналогично проверить `WheelOfVocation.tsx`.
- **Переполнение/наложение:** абсолютный таб-бар перекрывает заголовок пирамиды —
  выверить `pt-16`/отступы; длинные названия уровней (`truncate`) — дать перенос
  или уменьшить, чтобы влезали в контейнер.

**Критерий:** на светлой теме текст колеса/пирамиды читаем, кнопки в палитре,
ничего не наложено и не обрезано. Проверяет владелец (тестировщик) в обеих темах.

## Порядок и риск
1. Трек 1 (гейт B) — снимает первопричину, низкий риск.
2. Трек 3 (ярлык) — мелкий, зависит от 1.
3. Трек 4 (экстрактор интересов) — правка промпта, средний риск, изолирован.
4. Трек 2 (новый этап + перенумерация) — **самый рискованный** (63 ссылки на
   номера шагов + клиент + пирамида); делать последним, отдельным коммитом, с
   прогоном сквозного сценария.

**Риск-контроль:** правки коуч-потока чувствительны (см. [[feedback_careful-layout-edits]]);
после каждого трека — `tsc` + тесты + ручной сквозной прогон (экспресс → тесты →
глубокая: Цель→…→Барьеры→…→Финал → отчёт с барьерами и наполненной картой).

## Verification (сквозной сценарий)
- Экспресс-сессия завершена → предложена глубокая (не раньше).
- Глубокая идёт по 7 этапам, ярлыки пирамидные, счётчик не откатывается.
- Этап «Барьеры» задаёт вопрос про страхи/убеждения; ответ виден в отчёте.
- Колесо талантов наполняется по ходу (tech/startup зажигаются на релевантных репликах).
- `npx tsc --noEmit` и `npx vitest run` — зелёные.

## Примечание про деплой
Эти баги живут на **боевом сайте** (origin/main), независимо от несланных Треков
A–D этой сессии. После правок — решить с владельцем момент пуша/деплоя.
