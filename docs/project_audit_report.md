# Комплексный аудит платформы «МоёПризвание»
*Анализ архитектуры, дизайна, безопасности и технического долга проекта*

---

## 🏛️ Раздел 1. Архитектура и База данных

### Сильные стороны:
* **Next.js Standalone**: Сборка настроена в режиме `standalone` (в `next-config.mjs`), что снижает вес итогового бандла до минимума и делает приложение идеальным для serverless-хостинга.
* **Безопасные UUID**: Первичные ключи таблиц используют формат `UUID v4` на уровне СУБД, что исключает перебор ID злоумышленниками.
* **Таймзоны (Timestamptz)**: Все даты в БД хранятся с учетом временных зон, обеспечивая точную аналитику лидов.

### Обнаруженный технический долг:
1. **Отсутствие реферальных полей в Prisma**: В описании манифеста упоминается реферальная система, однако в схеме `prisma.schema` модель `User` не содержит явных полей `referredBy` или `referralCount`. Рефералы, скорее всего, учитываются через внешние механизмы или логируются отдельно, что усложняет аналитику внутри СУБД.
2. **In-memory Redis Cache**: Для сессий диагностики используется Redis. В случае падения инстанса Redis без настроенного snapshotting (RDB/AOF) прогресс тестов может быть утерян. Однако в проекте предусмотрено дублирование в PostgreSQL (`diagnosticAnswers` в модели `User`), что нивелирует этот риск.

## 🤖 Раздел 2. ИИ-интеграция и Диалоги (Нейрокоуч Роман)

### Сильные стороны:
* **Мгновенный старт (Токеносбережение)**: Первое сообщение коуча отдается из локального кэша без вызова API, что экономит токены и ускоряет первую отрисовку.
* **Контекстная глубина**: Передача истории последних 8 реплик диалога в ИИ-экстрактор предотвращает «забывание» ранее выявленных талантов школьника в процессе чата.
* **Безопасность генерации отчетов**: Внедренный fallback-механизм гарантирует, что даже при полном падении внешнего ИИ-интенса пользователь получит аккуратный отчет на основе его реальных ответов (RIASEC и Big Five), а не пустые плашки.

### Точки роста:
* **Лимиты API**: Использование Gemini/GPT-4o-mini требует мониторинга лимитов (RPM/TPM). При резком наплыве пользователей (например, во время школьного вебинара) API может возвращать ошибку 429. Fallback-система закроет эту проблему визуально, но рекомендуется внедрить очередь запросов (Queue) для генерации тяжелых ИИ-отчетов.

## 📐 Раздел 3. UX/UI и Дизайн-системы

### Сильные стороны:
* **Динамический сторителлинг**: Автовыбор тем тестов (`SPACE`, `CREATIVE`, `BUSINESS`) создает глубокое эмоциональное вовлечение подростка. Геймификация снижает процент отказов (bounce rate) на этапе тестирования.
* **Легковесный SVG-радар**: Использование самописного SVG для Радара RIASEC (вместо тяжелых библиотек Chart.js/Recharts) повышает скорость загрузки страницы и позволяет гибко управлять свечением через CSS.
* **Исключение мерцания тем**: Синхронный JS-скрипт в `<head>` считывает тему из `localStorage` до рендеринга страницы, предотвращая «белую вспышку» при входе на сайт в темном режиме.

### Точки роста:
* **Оптимизация видеофонов**: Файлы фоновых видео в `public/assets/videos` весят более 25-30 МБ. На мобильных устройствах с 3G/LTE это приведет к долгому старту первой отрисовки (LCP). Необходимо пережать видео с использованием кодека H.265/AV1 и уменьшить битрейт, сохранив качество (целевой вес файла — до 3-5 МБ).

## 🔒 Раздел 4. Безопасность и SEO

### Сильные стороны:
* **Token Exchange в ботах**: Вход из Telegram/MAX ботов защищен короткими одноразовыми токенами обмена (AuthExchangeToken), что предотвращает кражу сессий в веб-контейнерах мессенджеров.
* **Политики безопасности**: Внедрены заголовки `X-Frame-Options: DENY` (защита от кликджекинга) и `Permissions-Policy` (блокировка несанкционированного доступа к камере/микрофону).
* **SEO-оптимизация**: Внедрена семантическая разметка JSON-LD для Google/Яндекс и прописаны теги OpenGraph / Twitter Card для красивого отображения ссылок в соцсетях.

## 🛠️ Раздел 5. Рекомендации по оптимизации (План действий)

1. **Компрессия видео**: Сжать видеофоны `background-video.mp4` и `background-video-light.mp4` с помощью FFmpeg:
   ```bash
   ffmpeg -i input.mp4 -vcodec libx265 -crf 28 -tag:v hvc1 output.mp4
   ```
2. **Реферальные реляции**: Добавить реферальные поля напрямую в схему Prisma для прозрачного отслеживания амбассадоров на уровне базы данных.
3. **Lazy Loading 3D-компонентов**: Так как в `next.config.mjs` транслируются пакеты `three` (Three.js), необходимо убедиться, что все 3D-виджеты импортируются через `dynamic(() => import(...), { ssr: false })` и загружаются только тогда, когда пользователь доскроллил до них.
