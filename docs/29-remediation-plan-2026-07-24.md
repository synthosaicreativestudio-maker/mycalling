**Файлы.** `app/api/v1/coach/chat/route.ts` (ядро), `app/lib/coach/prompts.ts`,

### Шаг 1.1 — Сначала тест, который ловит баг (red)
1. Написать юнит-тест на **чистую функцию вычисления шага** (её выделю в 1.2). Кейс из
   скриншота: `{ fullName:'Роман', phone:null, age:'15' }` при `lastStep=2` → результат
   **не должен** быть «шаг 2 / подключить канал». Ожидаем шаг вперёд (Класс).
2. Кейс монотонности: если `stepReached=5`, а кандидат по флагам = 3 (ИИ «потерял» поле)
   → итог **не меньше 5**.
3. Кейс гостя без телефона: `phone=null`, но `fullName` есть → доходит до психоблоков,
   **никогда** не откатывается на «подключите канал».
4. Запустить — тесты падают (подтверждаем, что баг воспроизводится в логике).

### Шаг 1.2 — Выделить вычисление шага в чистую функцию + монотонность
1. Вынести из `route.ts:378-534` всю деривацию шага в чистую функцию
   `computeCandidateStep(extractedData): { step, currentStepBefore }` (без побочных
   эффектов, тестируемую).
2. Ввести **монотонный счётчик**: в `extractedData` добавить поле `stepReached` (макс.
   достигнутый шаг). Правило:
   `finalStep = Math.max(extractedData.stepReached ?? 0, candidate)` — **шаг не убывает**.
3. Продвижение вперёд разрешено **только когда текущий вопрос закрыт** (соответствующее
   поле извлечено). Т.е. флаги теперь **разрешают переход вперёд**, но физически не могут
   вернуть назад.
4. После вычисления: `extractedData.stepReached = finalStep` и `extractedData.currentStep = finalStep`
