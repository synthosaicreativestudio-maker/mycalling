# PRD v1 — МоеПризвание

## Цель MVP
Создать первый рабочий web MVP, который проводит диагностику, рассчитывает профиль ученика и выдает карьерный отчет с матчингом профессий, предметов и направлений обучения.

## KPI первого этапа
- 300 регистраций;
- 100 завершенных диагностик;
- completion rate диагностики выше 55%;
- не менее 20 качественных отзывов от родителей;
- не менее 15 заявок на консультацию.

## Основной сценарий
1. Пользователь заходит на лендинг.
2. Начинает единый семейный сценарий.
3. Оставляет контакт в начале.
4. Заполняет краткий контекстный профиль.
5. Проходит диагностику.
6. Получает итоговый отчет.
7. Выбирает следующий шаг.

## Scope MVP
### Входит
- лендинг;
- стартовый onboarding;
- capture контакта;
- профиль ученика;
- диагностика на 25–35 минут;
- скоринг результатов;
- топ профессий;
- предметы;
- направления обучения;
- единый отчет;
- parent summary;
- CTA на консультацию;
- сохранение результата в локальную сессию/профиль.

### Не входит
- вузы и колледжи;
- прогноз шансов поступления;
- AI-наставник;
- AI-репетитор;
- личный кабинет эксперта;
- платежи.

## Ключевые пользовательские результаты
### Для ученика
- понять свои сильные стороны;
- увидеть подходящие профессии;
- получить понятный язык без давления.

### Для родителя
- получить более серьезную, объяснимую картину;
- увидеть практические рекомендации;
- понять, что делать дальше.

## Ключевые продуктовые требования
- Результат должен быть объяснимым.
- Интерфейс должен быть простым и современным.
- Диагностика должна ощущаться легче, чем классический тест.
- Пользователь должен видеть прогресс внутри диагностики.
- После отчета обязательно должны быть следующие шаги.

## Ключевые CTA после отчета
- сохранить результат;
- вернуться позже;
- открыть персональный план развития;
- посмотреть предметы и направления;
- оставить заявку на консультацию.
