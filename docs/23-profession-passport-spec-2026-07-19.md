# Цифровой паспорт профессии — полная спецификация полей

**Дата:** 19 июля 2026
**Назначение:** ТЗ для наполнения каталога профессий (`app/data/professions_db.ts`).
Этот документ можно целиком отдать ИИ-генератору или человеку — здесь все поля,
типы, допустимые значения (наши реальные словари) и заполненный пример.

---

## 1. Зачем и главный принцип

У ученика есть **цифровой профиль** (собирается из тестов). Чтобы подбирать
профессии по всему профилю, а не по одним интересам, каждой профессии нужен
**симметричный паспорт** — что она требует. Подбор = сравнение двух паспортов по
одним осям.

**Главное правило: общий словарь.** Ценности, сильные стороны, способности
описываются на паспорте профессии **теми же кодами**, что и в профиле ученика —
иначе паспорта не сравнятся. Все словари — ниже, использовать ТОЛЬКО их.

---

## 2. Полный список полей

Легенда: **об.** = обязательное, **оп.** = опциональное.

| Поле | Тип | Об/Оп | Что означает |
|---|---|:--:|---|
| `id` | string (kebab-case, латиница) | об. | Уникальный технический ключ. Напр. `general-practitioner`. |
| `name` | string (рус.) | об. | Название профессии. Уникально. Напр. `Врач-терапевт`. |
| `industry` | string из §3.1 | об. | Отрасль/кластер ТОЛЬКО из списка отраслей. |
| `tier` | `everyday` \| `future` \| `dream` | об. | Слой каталога (баланс 70/20/10): массовая / трендовая / мечта-экзотика. |
| `archetype` | string (kebab-case) | об. | Ключ группировки родственных ролей. Все юристы → `lawyer`, все врачи → `doctor`. |
| `riasec` | массив 1–3 кодов из §3.2 | об. | Интересы (Holland-код), по убыванию значимости. «Хочу». |
| `gardner` | массив 1–3 кодов из §3.3 | об. | Типы интеллекта, которые задействует. «Таланты». |
| `bigFive.traits` | объект, §3.4 | об. | Черты характера под профессию (high/low). «Стиль». |
| `cognitiveDemand` | `low` \| `medium` \| `high` | **новое, об.** | Уровень требуемых способностей (сверяется с ICAR-band ученика). «Могу». |
| `values` | массив PVQ-кодов из §3.5 | **новое, об.** | Какие ценности профессия закрывает. «Ради чего». |
| `viaFit` | массив VIA-кодов из §3.6 | **новое, об.** | Какие сильные стороны характера задействует. |
| `subjects` | массив string (рус.) | об. | 3–4 школьных предмета РФ — «что учить сейчас». |
| `skills.hard` | массив string | об. | 3–4 профессиональных (hard) навыка. |
| `skills.soft` | массив string | об. | 2–3 гибких (soft) навыка. |
| `demand` | `high` \| `medium` \| `low` | об. | Реальный спрос на рынке РФ. Для конкурсных/редких — `low`. |
| `skillFormula` | массив string | об. | 2–3 переносимые компетенции (что останется, если роль изменится). |
| `transferableTo` | массив string | об. | 2–4 смежные профессии — честный «план Б». |
| `summary` | string (рус.) | об. | 1 предложение — суть профессии простым языком для подростка. |
| `why` | string (рус.) | об. | 1–2 предложения — кому подходит, через какие интересы/характер. |

> **Запрет по контенту (`why`/`summary`):** нельзя использовать аббревиатуру «ИИ»
> и словосочетание «искусственный интеллект» — заменять на «алгоритмы»,
> «цифровые системы», «автоматизация».

## 3. Контролируемые словари (использовать ТОЛЬКО эти значения)

```

Оркестрация ИИ и Агенты
Робототехника и хардвер
Космические технологии и коммерция
Инженерия и промышленность
Энергетика и эко-технологии
Биотехнологии и биоинженерия
Медицина и здравоохранение
Маркетинг, PR и бренд-менеджмент
Управление и бизнес-консалтинг
Финансовые технологии и инвестиции
Дизайн и креативные индустрии
Медиа, блогинг и контент-производство
Образование и EdTech
Психология и ментальное здоровье
Юриспруденция, право и безопасность
Фундаментальная наука и исследования
Агротехнологии и сити-фермерство
Транспорт, логистика и беспилотные системы
Спорт, фитнес и велнес
Туризм и премиальное гостеприимство
```
> Если профессия не влезает ни в одну — сначала добавить новую отрасль в
> `industries[]` (отдельным решением), а не выдумывать строку на месте.

### 3.2 RIASEC (riasec) — интересы, 6 кодов
```
Realistic       — практика, руки, техника
Investigative   — анализ, исследование, наука
Artistic        — творчество, эстетика
Social          — люди, помощь, обучение
Enterprising    — лидерство, продажи, влияние
Conventional    — порядок, данные, системность
```
Массив из 1–3 кодов по убыванию (первый — ведущий).

### 3.3 Gardner (gardner) — типы интеллекта, 8 кодов
```
Linguistic  Logical-Mathematical  Spatial-Visual  Bodily-Kinesthetic
Musical  Interpersonal  Intrapersonal  Naturalist
```

### 3.4 Big Five (bigFive.traits)
Объект, ключи и значения:
```
Openness | Conscientiousness | Extraversion | Agreeableness | Stability
значение каждого:  'high'  или  'low'   (только выраженные — 2–4 черты, не все 5)
```
> Stability = эмоциональная устойчивость (обратна невротизму).

### 3.5 Ценности PVQ (values) — 10 кодов
```
self_direction  — Самостоятельность
stimulation     — Новизна и риск
hedonism        — Удовольствие
achievement     — Достижения
power           — Влияние
security        — Безопасность и стабильность
conformity      — Следование правилам
tradition       — Традиции
benevolence     — Забота о близких
universalism    — Справедливость и забота о мире
```
Массив из 2–4 кодов — какие ценности реализует профессия.

### 3.6 Сильные стороны VIA (viaFit) — 24 кода
```
creativity  curiosity  judgment  love_of_learning  perspective
bravery  perseverance  honesty  zest
love  kindness  social_intelligence
teamwork  fairness  leadership
forgiveness  humility  prudence  self_regulation
appreciation_of_beauty  gratitude  hope  humor  spirituality
```
Массив из 2–5 кодов — какие сильные стороны задействует профессия.

## 4. Как поля соединяются с профилем ученика (для понимания)

| Ось паспорта профессии | С чем сравнивается у ученика |
|---|---|
| `riasec` | RIASEC-баллы (тест RIASEC) |
| `gardner` | типы интеллекта |
| `bigFive.traits` | Big Five (тест BFI) |
| `cognitiveDemand` | ICAR-band (developing/solid/strong ↔ low/medium/high) + COGNITIVE_STYLE |
| `values` | топ-ценности PVQ |
| `viaFit` | топ-5 сильных сторон VIA |

> НЕ входят в матчинг (это «как дойти», а не «куда»): настойчивость (GROWTH),
> прокрастинация, контекст/ресурсы. Их профиль использует в отчёте как совет,
> но профессии по ним не фильтруются.

## 5. Полный пример заполненного паспорта

```ts
{
  id: 'general-practitioner',
  name: 'Врач-терапевт',
  industry: 'Медицина и здравоохранение',
  tier: 'everyday',
  archetype: 'doctor',
  riasec: ['Social', 'Investigative', 'Conventional'],
  gardner: ['Interpersonal', 'Logical-Mathematical', 'Intrapersonal'],
  bigFive: { traits: { Conscientiousness: 'high', Agreeableness: 'high', Stability: 'high' } },
  cognitiveDemand: 'high',
  values: ['benevolence', 'security', 'achievement'],
  viaFit: ['kindness', 'judgment', 'perseverance', 'social_intelligence'],
  subjects: ['Биология', 'Химия', 'Русский язык'],
  skills: {
    hard: ['Диагностика заболеваний', 'Назначение лечения', 'Ведение медкарт', 'Неотложная помощь'],
    soft: ['Эмпатия и общение с пациентом', 'Внимательность', 'Стрессоустойчивость']
  },
  demand: 'high',
  skillFormula: ['Клиническое мышление', 'Забота о людях', 'Работа с большим объёмом данных'],
  transferableTo: ['Педиатр', 'Врач-кардиолог', 'Онлайн-терапевт (Теледоктор)', 'Медицинский эксперт'],
  summary: 'Врач-терапевт — первый доктор, к которому обращается человек: ставит диагноз, лечит и направляет к узким специалистам.',
  why: 'Подходит тем, кто хочет помогать людям (Social), любит разбираться в причинах (Investigative) и готов к ответственности и постоянной учёбе.'
}
```

## 6. Правила качества (обязательно проверять)

1. `riasec` — строго 1–3 кода, по убыванию значимости, без дублей.
2. `values`, `viaFit`, `gardner` — только коды из словарей §3, никаких своих строк.
3. `industry` — точное совпадение со строкой из §3.1.
4. `tier` — по правилу 70/20/10: массовые с большим спросом → `everyday`;
   растущие/технологичные → `future`; конкурсные/редкие/спорт/служение → `dream`.
5. `archetype` — родственные роли делят один ключ (все юристы `lawyer`, все
   инженеры-строители `civil-engineer` и т.д.), чтобы в отчёте сворачиваться.
6. Честность для мечты: `demand: 'low'` + реальный `transferableTo` (план Б).
7. Никакой «ИИ»/«искусственный интеллект» в текстах — только «алгоритмы» и т.п.

После наполнения — прогнать `npx vitest run app/data/professions_db.test.ts`
(валидатор поймает битые коды/отрасли/дубли).

---

## 7. Формат выдачи

Для каждой профессии — один объект строго по §2 (все обязательные поля).
Для машинной обработки лучше JSON-массив объектов; для вставки в код —
TypeScript-объекты как в §5. Наш скрипт `scripts/generate-profession.js` уже
выдаёт готовые TS-записи по этой схеме (кроме новых полей `cognitiveDemand`/
`values`/`viaFit` — их добавление в промпт генератора — часть текущей работы).
