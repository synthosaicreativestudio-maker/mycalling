# Инструкция по деплою и настройке сервера
## Проект: «МоеПризвание» (домен: `synthosai.ru`)

В этом документе описана архитектура деплоя приложения, процесс первоначальной настройки сервера, обновление версии сайта и настройка доменного имени с SSL (HTTPS).

---

## 1. Архитектура деплоя (Как всё реализовано)

Приложение разворачивается на виртуальном сервере (VPS) под управлением **Ubuntu** со следующей структурой взаимодействия компонентов:

```
[ Пользователь ] 
       │ (порт 80 / 443 - HTTPS)
       ▼
  [ Nginx ] (Reverse Proxy, SSL-терминация через Let's Encrypt)
       │ (проксирование на локальный порт 3000)
       ▼
 [ Next.js App ] (Запущено в Standalone-режиме: server.js)
       ▲
       │ (управление процессом, автозапуск при перезагрузке)
   [ PM2 ]
```

### Ключевые компоненты:
1. **GitHub репозиторий:** `https://github.com/synthosaicreativestudio-maker/mycalling.git` является единственным источником правды для кода.
2. **Next.js Standalone Mode:** В `next.config.mjs` настроен вывод сборки в виде изолированного сервера (`output: 'standalone'`). Это позволяет запускать приложение без необходимости держать тяжелую папку `node_modules` на сервере, экономя оперативную память (Next.js упаковывает только используемые зависимости).
3. **PM2 (Process Manager):** Менеджер процессов Node.js. Запускает приложение в фоне, следит за логами, автоматически перезапускает его при сбоях (crash recovery) и обеспечивает запуск сервера после перезагрузки VPS (через systemd).
4. **Nginx:** Служит веб-сервером перед приложением (Reverse Proxy). Он принимает входящие HTTPS-запросы с домена `synthosai.ru`, расшифровывает SSL и перенаправляет их локально на `localhost:3000`.

## 2. Скрипты автоматизации в проекте

В папке `deploy/` подготовлены два скрипта для работы с сервером (IP сервера: `111.88.145.206`, пользователь: `ubuntu`):

### А. [deploy/setup-server.sh](file:///Users/verakoroleva/Desktop/%D0%B8%D0%B4%D0%B5%D0%B8/%D0%9C%D0%BE%D0%B5%20%D0%9F%D1%80%D0%B8%D0%B7%D0%B2%D0%B0%D0%BD%D0%B8%D0%B5/deploy/setup-server.sh)
Служит для **первоначальной настройки** чистого сервера.
* Обновляет системные пакеты (`apt update & upgrade`).
* Устанавливает **Node.js 20** и глобальный менеджер процессов **PM2**.
* Клонирует репозиторий в `/home/ubuntu/mycalling`.
* Производит чистую сборку проекта (`npm install && npm run build`).
* Копирует статические файлы (`public`, `.next/static`) в директорию `.next/standalone` для работы в изолированном режиме.
* Регистрирует и запускает службу PM2 под именем `mycalling` на порту `3000`.

### Б. [deploy/update.sh](file:///Users/verakoroleva/Desktop/%D0%B8%D0%B4%D0%B5%D0%B8/%D0%9C%D0%BE%D0%B5%20%D0%9F%D1%80%D0%B8%D0%B7%D0%B2%D0%B0%D0%BD%D0%B8%D0%B5/deploy/update.sh)
Служит для **быстрого наката обновлений** (после того как вы отправили изменения в ветку `main` на GitHub).
* Делает `git pull origin main` на сервере.
* Доустанавливает новые зависимости через `npm install`.
* Собирает проект заново и переносит статические файлы.
* Перезапускает процесс в PM2 без простоя (`pm2 restart mycalling`).

## 3. Инструкция: Как запустить сайт на сервере с нуля

### Шаг 1. Подключение к серверу по SSH
Откройте терминал на компьютере и выполните команду:
```bash
ssh ubuntu@111.88.145.206
```

### Шаг 2. Запуск скрипта первоначальной настройки
Если скрипта еще нет на сервере, скопируйте его туда или создайте файл. Самый простой способ скопировать папку `deploy` с локального компьютера на сервер:
```bash
scp -r deploy/ ubuntu@111.88.145.206:/home/ubuntu/
```
Затем на сервере запустите его:
```bash
bash /home/ubuntu/deploy/setup-server.sh
```
После завершения приложение запустится локально на порту `3000`. Проверить это можно, набрав в браузере: `http://111.88.145.206:3000`.

## 4. Связка домена (synthosai.ru) и SSL через Nginx

Чтобы сайт открывался по красивому домену `synthosai.ru` (по HTTPS), выполните на сервере следующие настройки веб-сервера Nginx.

### Шаг 1. Установка Nginx и Certbot (для SSL)
На сервере выполните:
```bash
sudo apt update
sudo apt install nginx certbot python3-certbot-nginx -y
```

### Шаг 2. Создание конфигурации Nginx для домена
Создайте файл конфигурации:
```bash
sudo nano /etc/nginx/sites-available/synthosai.ru
```
Вставьте следующий блок конфигурации (Proxy pass на порт 3000):
```nginx
server {
    listen 80;
    server_name synthosai.ru www.synthosai.ru;

    # Максимальный размер загружаемых файлов (например, для лид-форм)
    client_max_body_size 20M;

    location / {
        proxy_pass http://localhost:3000;
        proxy_http_version 1.1;
        proxy_set_header Upgrade $http_upgrade;
        proxy_set_header Connection 'upgrade';
        proxy_set_header Host $host;
        proxy_cache_bypass $http_upgrade;
        
        # Передача реальных IP-адресов пользователей в Next.js
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto $scheme;
    }
}
```
Сохраните файл (`Ctrl + O`, затем `Enter`) и выйдите (`Ctrl + X`).

### Шаг 3. Активация конфигурации и перезапуск Nginx
```bash
# Создаем символическую ссылку для активации сайта
sudo ln -s /etc/nginx/sites-available/synthosai.ru /etc/nginx/sites-enabled/

# Проверяем конфигурацию Nginx на ошибки
sudo nginx -t

# Перезапускаем веб-сервер
sudo systemctl restart nginx
```

### Шаг 4. Получение SSL-сертификата (HTTPS)
Запустите Certbot, он автоматически найдет конфигурацию Nginx, выпустит бесплатный SSL-сертификат Let's Encrypt и настроит перенаправление с HTTP на HTTPS:
```bash
sudo certbot --nginx -d synthosai.ru -d www.synthosai.ru
```
*Certbot задаст вопрос об Email (укажите ваш) и попросит согласиться с условиями. После этого сайт станет доступен по адресу **`https://synthosai.ru`**.*

## 5. Инструкция по обновлению кода (После пуша на GitHub)

Когда вы внесли изменения на локальном компьютере, сделали коммит и отправили его на GitHub (`git push`):

Вы можете запустить обновление сайта удаленно одной командой с локального компьютера (не заходя на сервер вручную):
```bash
ssh ubuntu@111.88.145.206 'bash /home/ubuntu/mycalling/deploy/update.sh'
```

Эта команда зайдет на сервер, скачает изменения с GitHub, пересоберет проект Next.js и перезапустит процесс PM2. Сайт обновится за 15-30 секунд.

## 6. Полезные команды для администрирования на сервере

Если вам нужно проверить состояние приложения на сервере, подключитесь по SSH и используйте следующие команды:

* **Мониторинг запущенных процессов:**
  ```bash
  pm2 status
  ```
* **Просмотр логов в реальном времени (ошибки, консоль вывода):**
  ```bash
  pm2 logs mycalling
  ```
* **Перезапуск приложения вручную:**
  ```bash
  pm2 restart mycalling
  ```
* **Просмотр логов веб-сервера Nginx:**
  ```bash
  sudo tail -f /var/log/nginx/error.log
  ```
