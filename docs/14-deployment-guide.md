## 1. Архитектура деплоя и ключи доступа

Приложение разворачивается на виртуальном сервере (VPS) под управлением **Ubuntu** со следующей структурой взаимодействия компонентов:

```
[ Пользователь ] 
       │ (порт 80 / 443 - HTTPS)
       ▼
  [ Nginx ] (Reverse Proxy, SSL-терминация через Let's Encrypt)
       │ (проксирование на локальный порт 3000)
       ▼
 [ Next.js App ] (Запущено в Standalone-режиме: server.js)
       ▲
       │ (управление процессом, автозапуск при перезагрузке)
   [ PM2 ]
```

### Доступы и цепочка ключей (Данные из vpn/docs/CREDENTIALS.md):

1. **Американский сервер (US Server - транзитный бастион):**
   * **IP:** `37.1.212.51`
   * **Пользователь:** `root`
   * **Пароль для входа:** `LEJ6U5chSK`
   
2. **Российский сервер (Яндекс ВМ - Relay RU / Целевой сервер проекта):**
   * **IP:** `111.88.145.206`
   * **Пользователь:** `ubuntu`
   * **Способ авторизации:** по SSH-ключу `ed25519`, который лежит на US-сервере (в директории `root/.ssh/`).
   
3. **Цепочка подключения к целевой машине:**
   * Сначала подключаемся к US серверу: `ssh root@37.1.212.51` (пароль `LEJ6U5chSK`).
   * С US сервера подключаемся к целевому серверу: `ssh ubuntu@111.88.145.206` (ключ подхватывается автоматически).

---
3. **PM2 (Process Manager):** Менеджер процессов Node.js. Запускает приложение в фоне, следит за логами, автоматически перезапускает его при сбоях (crash recovery) и обеспечивает запуск сервера после перезагрузки VPS (через systemd).
4. **Nginx:** Служит веб-сервером перед приложением (Reverse Proxy). Он принимает входящие HTTPS-запросы с домена `synthosai.ru`, расшифровывает SSL и перенаправляет их локально на `localhost:3000`.

## 4. Связка домена (synthosai.ru) и SSL через Nginx

Чтобы сайт открывался по красивому домену `synthosai.ru` (по HTTPS), выполните на сервере следующие настройки веб-сервера Nginx.

### Шаг 1. Установка Nginx и Certbot (для SSL)
На сервере выполните:
```bash
sudo apt update
sudo apt install nginx certbot python3-certbot-nginx -y
```

### Шаг 2. Создание конфигурации Nginx для домена
Создайте файл конфигурации:
```bash
sudo nano /etc/nginx/sites-available/synthosai.ru
```
Вставьте следующий блок конфигурации (Proxy pass на порт 3000):
```nginx
server {
    listen 80;
    server_name synthosai.ru www.synthosai.ru;

    # Максимальный размер загружаемых файлов (например, для лид-форм)
    client_max_body_size 20M;

    location / {
        proxy_pass http://localhost:3000;
        proxy_http_version 1.1;
        proxy_set_header Upgrade $http_upgrade;
        proxy_set_header Connection 'upgrade';
        proxy_set_header Host $host;
        proxy_cache_bypass $http_upgrade;
        
        # Передача реальных IP-адресов пользователей в Next.js
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto $scheme;
    }
}
```
Сохраните файл (`Ctrl + O`, затем `Enter`) и выйдите (`Ctrl + X`).

### Шаг 3. Активация конфигурации и перезапуск Nginx
```bash
# Создаем символическую ссылку для активации сайта
sudo ln -s /etc/nginx/sites-available/synthosai.ru /etc/nginx/sites-enabled/

# Проверяем конфигурацию Nginx на ошибки
sudo nginx -t

# Перезапускаем веб-сервер
sudo systemctl restart nginx
```

### Шаг 4. Получение SSL-сертификата (HTTPS)
Запустите Certbot, он автоматически найдет конфигурацию Nginx, выпустит бесплатный SSL-сертификат Let's Encrypt и настроит перенаправление с HTTP на HTTPS:
```bash
sudo certbot --nginx -d synthosai.ru -d www.synthosai.ru
```
*Certbot задаст вопрос об Email (укажите ваш) и попросит согласиться с условиями. После этого сайт станет доступен по адресу **`https://synthosai.ru`**.*

## 6. Полезные команды для администрирования на сервере

Эти команды выполняются **после входа на российский сервер (`111.88.145.206`)**:

* **Мониторинг запущенных процессов (проверка работы сайта):**

  ```bash
  pm2 logs mycalling
  ```

  ```bash
  pm2 restart mycalling
  ```

  ```bash
  sudo tail -f /var/log/nginx/error.log
  ```
