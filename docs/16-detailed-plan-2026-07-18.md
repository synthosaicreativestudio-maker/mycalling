## Этап 0. Фундамент: рефакторинг движка диагностики

Без этого этапа каждый новый тест (VIA, PVQ) будет раздувать `next-question/route.ts` (сейчас 522 строки, скоринг зашит в роут, правильные ответы ICAR захардкожены по `id` на строках 95–97).

### 0.1. Вынос скоринга в `app/lib/diagnostic/scoring.ts`

```ts
export interface TestScorer {
  testCode: string;
  score(answers: Record<string, number>, questions: DiagnosticQuestion[]): {
    scores: Record<string, number>;
    reliability: 'high' | 'medium' | 'low';
  };
}
```

- Реализовать скореры `riasecScorer`, `bfiScorer`, `icarScorer`, `procrastinationScorer`, перенеся логику из `next-question/route.ts:70-112` один в один (без изменения формул — этап 0 не меняет поведение).
- Реестр: `export const scorers: Record<string, TestScorer>`.
- В роуте цикл по `scorers` вместо четырёх веток if/else; `createMany` собирается из результатов реестра.

### 0.2. Правильные ответы — в данные вопросов

- В `DiagnosticQuestion` (`app/data/questions.ts:9-17`) добавить опциональные поля:
  - `correctValue?: number` — для ICAR-заданий (перенести из хардкода route.ts:95-97);
  - `subscale?: string` — уже есть как `scale`, оставить `scale`, но задокументировать значения по тестам.
- `icarScorer` считает по `correctValue`, а не по списку id.

### 0.3. Вычисляемый `reliability` — `app/lib/diagnostic/reliability.ts`

```ts
export function computeReliability(answers, questions): 'high'|'medium'|'low'
```
Правила (единые для всех likert-тестов):
- «плоский паттерн»: ≥90% одинаковых ответов → `low`;
- противоречия в обратных парах (`reverseScored`): среднее расхождение >2.5 балла → понижение на одну ступень;
- доля пропусков >20% → понижение на одну ступень.
Заменить статические литералы `'high'`/`'medium'` в `next-question/route.ts:133-154`.

**Критерии приёмки этапа 0:** результаты RIASEC/BFI/ICAR/прокрастинации при одинаковых ответах идентичны текущим (снять эталонные фикстуры до рефакторинга); роут < 300 строк; новый тест добавляется без правки роута (вопросы + скорер).

**Оценка:** 1–2 дня. **Зависимости:** нет. Блокирует Д-1, Д-3, Д-5, Д-11.

## Д-2. Индекс согласованности — триангуляция (P0)

**Цель:** алгоритмическое «столкновение» данных коуча и тестов.

### Изменения

1. **Новый модуль `app/lib/profile/consistency.ts`**

```ts
export interface Contradiction {
  code: string;            // 'extraversion-vs-teamwork', 'riasec-vs-dream', ...
  testFact: string;        // «Экстраверсия 28%»
  coachFact: string;       // «Говорил, что обожает командную работу»
  probe: string;           // готовый фокусный вопрос от Романа
  weight: number;          // 1..3
}
export interface ConsistencyResult {
  index: number;           // 0..100
  level: 'high' | 'medium' | 'low';   // ≥75 / 50–74 / <50
  contradictions: Contradiction[];
}
export function computeConsistency(
  summary: SummaryProfile,          // riasec, bigFive, via, procrastination
  coachExtracted: Record<string, any>
): ConsistencyResult
```

2. **Правила сравнения (MVP — 5 детерминированных правил):**
   - `E`(BFI) < 2.5 при упоминании в `dreams`/`values`/`deepIdentity` слов из словаря «команда, общение, сцена, блогер…» → противоречие (и зеркально E > 3.5 + «люблю один, тишина»);
   - `C`(BFI) < 2.5 при `deepActions`/`values` со словами «дисциплина, режим, каждый день» → противоречие;
   - балл прокрастинации > 14 при заявленном «первом шаге прямо сейчас» (`deepFirstStep` непустой) → мягкое противоречие (weight 1);
   - ведущий RIASEC-код не пересекается с категорией мечты (`dreams`→маппинг ключевых слов на R/I/A/S/E/C в `app/data/dreamCategories.ts`) → противоречие;
   - VIA топ-5 не содержит `leadership`/`teamwork` при `deepIdentity` «лидер, капитан, организатор» → противоречие.
   - Словарный матчинг синхронный и дешёвый; ИИ-уточнение (сравнение смыслов через `generateJson`) — вторым слоем за фиче-флагом `CONSISTENCY_AI`, с фолбэком на словарь.
   - `index = 100 − Σ(weight × 12)`, минимум 0.

3. **Интеграция**
   - Вызов после сборки `summaryProfile` в `next-question/route.ts` (и в `generate-report/route.ts`); результат кладётся в `DigitalProfile.summary.consistency`.
   - Промпт отчёта: при `level !== 'high'` — обязательный раздел `innerConflicts: [{ title, text }]` «Внутренние противоречия — твои скрытые ресурсы», где каждый элемент строится из `probe`; при `high` — бейдж «Высокая достоверность профиля» в шапке отчёта (`app/report`).

4. **Тесты** — `app/lib/profile/consistency.test.ts`: фикстуры «полное согласие» (index ≥ 75, 0 противоречий), «интроверт-командник» (ловится правило 1), «прокрастинатор с первым шагом» (weight 1, index ≥ 75 при отсутствии других).

**Критерии приёмки:** см. тесты; в отчёте раздел появляется/исчезает по уровню; функция чистая, без обращений к БД.

**Оценка:** 2–3 дня. **Зависимости:** Д-1 (правило VIA — опционально, можно запустить с 4 правилами до VIA).
