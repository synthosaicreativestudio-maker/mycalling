## P1 — заметно пользователю, делать в первую очередь

### 1.1 Высота экрана коуча в оконном режиме (из скриншот-отчёта)
Поле ввода уходит под край окна / док macOS при малой высоте окна.
- `app/coach/page.tsx:648`: `h-screen` → `h-[100dvh]` (динамический viewport);
  паддинг `pt-28` пересмотреть под фактическую высоту sticky-шапки.
- Убедиться, что скроллится только внутренний блок истории чата, а строка ввода
  прижата к низу видимой области на всех брейкпоинтах (320px–desktop, включая
  «низкие» окна 1280×600).
- Проверка: Chrome DevTools, эмуляция + ручное окно уменьшенной высоты.

### 1.2 Speed Index 8.9s на мобильном лендинге
Причина — тяжёлый анимированный фон (DarkCosmicBackground + framer-motion каскады).
- Отключить/упростить фоновые анимации на мобильных (`prefers-reduced-motion` +
  media-query по ширине): статический градиент вместо анимированного шума.
- Отложить (lazy/dynamic import) framer-motion секции ниже первого экрана.
- Цель: Speed Index ≤ 4s mobile, Performance ≥ 92.

### 1.3 Контраст и читаемость кабинета коуча в светлой теме (из скриншот-отчёта)
- Кнопки «На главную»/«Заново» (`page.tsx` шапка): `bg-white/5 text-white/70` не видны
  в светлой теме → перевести на CSS-переменные или добавить `[data-theme="light"]`
  оверрайды в `globals.css`.
- Рёбра пирамиды в светлой теме: `stroke` → `var(--text-muted)`, поднять opacity
  незаполненных полигонов.
- Подпись «Наведите на уровень пирамиды…» → `var(--text-secondary)`.
- Тёмная тема: неактивная вкладка «Пирамида целей» сливается с подложкой → поднять
  контраст неактивного состояния (текст ≥ 4.5:1 на фоне переключателя).
- Модалка Колеса: подписи шкал вылезают за границы карточки → ограничить ширину SVG,
  увеличить внутренние паддинги, длинные подписи переносить/обрезать.

### 1.4 Симметрия карточек чата и визуализации (из скриншот-отчёта)
Левая колонка (чат) — `.glass-card` на странице, правая — фон внутри
`WheelOfVocation`/`PyramidOfAlignment`. Вынести обёртку-карточку на уровень
`page.tsx` для обеих колонок (одинаковые классы и `h-full`), графики центрировать
внутри. Результат: одинаковая высота и рамки обеих карточек на десктопе.

## P2 — методология и качество данных

### 2.1 Колесо талантов «мертвеет» в DEEP-фазе
Во время глубокой фазы экстрактор извлекает только deep-поля и не обновляет
`talentScores`. Добавить в DEEP-экстракцию опциональное обновление talentScores
(та же ratchet-логика Math.max), чтобы колесо продолжало жить во второй половине
сессии.

### 2.2 Валидность «архетипа по Юнгу»
Сейчас архетип определяется LLM «на глазок» в финальном промпте. Варианты:
- (быстро) пометить в отчёте как «гипотезу наставника», не как измерение;
- (правильно) вывести архетип детерминированно из уже собираемых VIA-сил и
  PVQ-ценностей по фиксированной таблице соответствия — данные уже есть в БД.

### 2.3 Триангуляция коучинг ↔ тесты
Противоречия (например, коуч видит предпринимателя, RIASEC говорит «исследователь»)
никак не подсвечиваются. Минимальный шаг: в финальном отчёте добавить блок
«Что сошлось / что расходится» — сравнение RIASEC-кода из тестов с talentScores
из коуч-сессии; расхождения формулировать как зону для обсуждения, не как ошибку.
