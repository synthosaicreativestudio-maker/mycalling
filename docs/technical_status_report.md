# Технический отчет о реализации платформы «МоёПризвание»
## Статус разработки систем, архитектуры и интеграций (MVP)

Настоящий документ содержит детальное описание реализованной технической архитектуры, модулей и внешних интеграций платформы «МоёПризвание». Все описанные компоненты спроектированы, разработаны и функционируют в рамках единого репозитория по модели бережливой соло-разработки.

---

## 1. СУБД и Модель данных (Бэкенд-ядро)

В качестве фундамента платформы используется реляционная база данных PostgreSQL, взаимодействие с которой осуществляется через Prisma ORM. Архитектура СУБД спроектирована с учетом гибкого разделения гостевого и авторизованного доступа.

### Реализованные модели данных:
* **User & Session (Better Auth)**: Хранение данных пользователей и активных сессий авторизации. Поддерживается одновременная запись стандартных и безопасных кук сессий (`better-auth.session_token` и `__secure-better-auth.session_token`).
* **VocationProfile**: Цифровой профиль подростка. Содержит качественные характеристики (мечты, барьеры, ценности), извлеченные из диалога с наставником.
* **DiagnosticResult**: Хранилище результатов тестирования. Содержит сырые ответы и рассчитанные по шкалам баллы (RIASEC, Big Five, ICAR, Лэй).
* **AuthLinkCode**: Таблица временных кодов связывания (`linkCode`), сопоставляющих гостевую веб-сессию с аккаунтами пользователей в мессенджерах.

---

## 2. Диалоговая система (Цифровой наставник Роман)

Модуль интерактивного взаимодействия, заменяющий классическое анкетирование и собирающий первичный качественный профиль подростка.

### Технический стек и логика:
* **Интеграция с API**: Прямое подключение к Google Gemini API на стороне сервера для динамической генерации реплик наставника.
* **16-этапный сценарий**: Реализована жесткая контекстная логика ведения диалога (от знакомства и выявления интересов до разбора барьеров и перехода к тестам).
* **Безопасность диалога (Prompt Engineering)**: Внедрены системные инструкции, блокирующие попытки пользователя перехватить управление моделью (системный джейлбрейк) или спровоцировать эхо-ответы.
* **Визуализация (Колесо Призвания)**: Интерактивный фронтенд-виджет WheelOfVocation (SVG/CSS), который динамически перестраивается и анимируется на основе выявляемых в диалоге склонностей подростка.

---

## 3. Модуль интерактивной диагностики

Комплекс тестов, предназначенный для количественной оценки способностей и характера подростка.

### Состав диагностического блока (29 вопросов):
1. **RIASEC (Голландская шкала)**: Определение типа профессиональной личности (реалистический, интеллектуальный, артистический, социальный, предприимчивый, конвенциональный).
2. **Big Five (Большая пятерка)**: Экспресс-оценка личностных качеств (экстраверсия, доброжелательность, добросовестность, эмоциональная стабильность, открытость опыту).
3. **ICAR (Когнитивные пробы)**: Оценка пространственного и логического мышления через интерактивные мини-задачи.
4. **Поведение (Методика Лэй)**: Выявление склонности к прокрастинации и волевых качеств.

### Фронтенд-инженерия тестов:
* **Zustand Store**: Централизованное управление локальным состоянием прохождения диагностики без лишних запросов к БД на каждом вопросе.
* **Click-Speed Lock**: Защита от бездумного прокликивания. Кнопки временно блокируются при слишком высокой скорости нажатия, предотвращая порчу диагностических данных.
* **Офлайн-режим (Offline-Resilience)**: При временном сбое интернет-соединения прогресс сохраняется в LocalStorage и синхронизируется с сервером при восстановлении сети.

## 4. Система бесшовной авторизации (AuthLink)

Технология беспарольного входа и связывания гостевого прогресса с постоянным аккаунтом через мессенджеры.

### Реализованные интеграции:
* **Telegram Bot API**: Бот авторизации и отправки уведомлений.
* **МАКС API (TamTam API)**: Интеграция с мессенджером МАКС для бесшовного подключения корпоративных и региональных пользователей.
* **Better Auth (TypeScript)**: Интеграция с бэкенд-авторизацией на стороне Next.js.
* **Silent Callback Flow**: Специальный вебхук-эндпоинт `/api/auth/telegram/callback`. При получении `linkCode` от бота бэкенд автоматически генерирует сессионный токен Better Auth, производит редирект и записывает куки в браузере, исключая необходимость ручного ввода логинов/паролей.

## 5. Личный кабинет и Модуль отчетности

Интерфейс предоставления результатов диагностики подросткам и их родителям.

### Страницы личного кабинета:
* **Вкладка «Карта талантов»**: Интерактивные графики распределения способностей и черт характера.
* **Вкладка «Карьерные рекомендации»**: Динамический список подходящих профессиональных областей с требованиями к навыкам и предметам ЕГЭ.
* **Вкладка «Родительский кабинет»**: Раздел для родителей с советами психологов по поддержке ребенка на этапе выбора.
* **Экспорт в PDF**: Специальный CSS-шаблон печати (`@media print`), скрывающий навигацию и оптимизирующий дашборд под стандартные листы A4 для печати или сохранения PDF в один клик без сторонних библиотек рендеринга.
