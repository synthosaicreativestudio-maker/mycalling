## 2. Ключевые решения по UX & Инженерии

### 2.1. Динамический Theme & Skin Engine (Cyberpunk / Anime / Minimal)

Во время первого общения в **🎯 Коуч Express** Роман между делом выявляет визуальные предпочтения школьника:
> *«В какой атмосфере тебе комфортнее общаться — футуристичный Киберпанк, яркий Аниме-стиль или чистый современный Минимализм?»*

Выбор сохраняется в поле `user.uiTheme` (`'cyberpunk' | 'anime' | 'minimal'`). Система мгновенно переключает **CSS-скин приложения** без переписывания сюжета или тестов:

| Скин (Тема) | Визуальные атрибуты | Цветовая гамма | Шрифты & UI-звуки |
|-------------|---------------------|----------------|-------------------|
| **CyberNeon (Cyberpunk)** | Неоновые рамки, сетка, HUD-элементы, темный фоновый шум | Фиолетовый / Неоновый Циан / Графит | Моноширинный HUD-шрифт, электронный UI-клик |
| **Anime Sci-Fi** | Сочные градиенты, светлое стекло (Glassmorphism), мягкие тени | Яркий лазурный / Розовый / Белый | Закругленный аккуратный sans-serif, мягкий клик |
| **Modern Minimal** | Премиальный лаконичный стиль (эстетика Apple / Headspace) | Тёмный графит / Тёплый беж / Золото | Изящный sans-serif, минималистичный поп-звук |

### 2.2. Интерактивная RPG-Карта узлов (Skill Tree + Fog of War)

Вместо скучного списка из 12 карточек школьник видит **интерактивную 2D-карту «Древо Познания»**:

```
                          ┌─────────────────────┐
                          │ 🎯 Коуч Express     │  (Узел 1: Старт)
                          └──────────┬──────────┘
                                     │
                  ┌──────────────────┴──────────────────┐
                  ▼                                     ▼
      ┌─────────────────────┐               ┌─────────────────────┐
      │ 🧭 Мир Интересов    │  (ВЫБОР A)    │ 🧠 Профиль Личности │  (ВЫБОР B)
      └──────────┬──────────┘               └──────────┬──────────┘
                 │                                     │
        ┌────────┴────────┐                   ┌────────┴────────┐
        ▼                 ▼                   ▼                 ▼
  ┌───────────┐     ┌───────────┐       ┌───────────┐     ┌───────────┐
  │ 💎 Таланты│     │ 🎭 Архети │       │ 🧩 Когни- │     │ 🌍 Ценнос-│
  │   (VIA)   │     │    пы     │       │   тивка   │     │    ти     │
  └───────────┘     └───────────┘       └───────────┘     └───────────┘
        ░                 ░                   ░                 ░
   [🌫️ ТУМАН ]       [🌫️ ТУМАН ]         [🌫️ ТУМАН ]       [🌫️ ТУМАН ]
```

**Механика карты:**
1. **Старт:** На карте активен только 1-й центральный узел — **🎯 Коуч Express**.
2. **Нелинейный выбор:** После прохождения коучинга рассеивается туман над **2–3 соседними узлами** (например: *«Мир Интересов»* vs *«Профиль Личности»*).
3. **Автономия:** Школьник сам решает, какую ветку исследовать первой.
4. **Туман войны (Fog of War):** Дальнейшие узлы закрыты туманом с замками. Они открываются по мере прохождения связующих ветвей.

---

### 2.3. Защита психометрической валидности (Психометрический Иммунитет)

Чтобы сюжет новеллы не исказил научно валидированные формулировки тестов:
- **Персонаж (Роман / Астра)** даёт **эмпатичную контекстную реплику**:
  *«У каждого из нас свой темп. Давай посмотрим, как ты действуешь в условиях неопределенности...»*
- **Плашка выбора (Choice Panel)** содержит **строгую калиброванную формулировку пункта** (BFI, RIASEC, VIA), оформленную как решение или внутренняя мысль протагониста.

---

### 2.4. Высокая производительность и ленивая загрузка (Performance Strategy)

1. **2D Canvas / Lighweight DOM для Brain Games:**
   Когнитивные задачи (N-Back на память, Flanker на внимание) верстаются на HTML5 Canvas / векторе без тяжелых 3D-движков. Размер виджета: **всего 50–100 Кб**!
2. **Dynamic Imports (`next/dynamic`):**
   Код последующих глав загружается в браузер **только при клике** на соответствующий узел карты.
3. **Фоновая подгрузка (Background Prefetching):**
   Пока пользователь проходит активную главу, Service Worker в фоновом режиме бесшумно скачивает ассеты 2 ближайших открытых узлов. Задержка перехода — **0 миллисекунд**.

## 4. Схема БД и Компонентов

### 4.1. Дополнения в schema.prisma

```prisma
// Расширение User: тема UI
model User {
  // ...
  uiTheme       String    @default("cyberpunk") @map("ui_theme") // 'cyberpunk' | 'anime' | 'minimal'
  // ...
}

// ── Прогресс по главам новеллы ──
model ModuleProgress {
  id          String    @id @default(dbgenerated("gen_random_uuid()")) @db.Uuid
  userId      String    @map("user_id") @db.Uuid
  user        User      @relation(fields: [userId], references: [id], onDelete: Cascade)
  moduleCode  String    @map("module_code")
  status      String    @default("NOT_STARTED") // 'NOT_STARTED' | 'IN_PROGRESS' | 'COMPLETED'
  progress    Int       @default(0)             // 0-100%
  answers     Json?                             // промежуточные ответы в новелле
  scores      Json?                             // финальные скоры
  insight     Json?                             // микро-инсайт
  startedAt   DateTime? @map("started_at") @db.Timestamptz(6)
  completedAt DateTime? @map("completed_at") @db.Timestamptz(6)
  createdAt   DateTime  @default(now()) @map("created_at") @db.Timestamptz(6)

  @@unique([userId, moduleCode])
  @@index([userId])
  @@map("module_progress")
}

// ── Архетипы: полный transcript с Зеркалом ──
model ArchetypeSession {
  id            String        @id @default(dbgenerated("gen_random_uuid()")) @db.Uuid
  userId        String        @unique @map("user_id") @db.Uuid
  user          User          @relation(fields: [userId], references: [id], onDelete: Cascade)
  status        SessionStatus @default(IN_PROGRESS)
  transcript    Json          // история диалога
  extractedData Json          // { dominant, secondary, shadow, aspirational, fullProfile }
  completedAt   DateTime?     @map("completed_at") @db.Timestamptz(6)
  createdAt     DateTime      @default(now()) @map("created_at") @db.Timestamptz(6)

  @@map("archetype_sessions")
}

// ── Рекомендации в ЛК ──
model Recommendations {
  id          String   @id @default(dbgenerated("gen_random_uuid()")) @db.Uuid
  userId      String   @unique @map("user_id") @db.Uuid
  user        User     @relation(fields: [userId], references: [id], onDelete: Cascade)
  professions Json?    // [{name, score, formula, whyMatch}]
  education   Json?    // {courses, universities, youtube, olympiads}
  plan90days  Json?    // {skills: [{name, action}], firstStep}
  superpowers Json?    // {archetype, viaTop5, hollandCode}
  warnings    Json?    // {shadow, procrastination, beliefs}
  conflicts   Json?    // [{testFact, coachFact, insight}]
  coverage    Float    @default(0)
  updatedAt   DateTime @updatedAt @map("updated_at") @db.Timestamptz(6)

  @@map("recommendations")
}
```
