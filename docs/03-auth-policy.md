# Политика авторизации и безопасности ЛК

В системе реализован строгий раздельный контур для создания новых учетных записей (регистрации) и входа в личный кабинет (авторизации).

## Ключевые правила

1. **Регистрация только через диалог**:
   - Новые аккаунты создаются исключительно в процессе общения с наставником Романом на [главной странице](file:///Users/verakoroleva/Desktop/идеи/Мое%20Призвание/app/page.tsx).
   - В момент указания номера телефона на Этапе 1 бэкенд нативно генерирует гостевую учетную запись, связывая с ней сессию.

2. **Запрет регистрации через кнопку ЛК**:
   - Страница `/auth` («Личный кабинет») предназначена **только для авторизации** существующих пользователей.
   - Если незарегистрированный пользователь пытается войти через Telegram-виджет или МАКС-бот, бэкенд-фильтр в `/api/auth/telegram` пресекает создание нового аккаунта, перенаправляет обратно на `/auth` с ошибкой `error=register_denied`.
   - В интерфейсе выводится предупреждающий баннер с предложением начать сессию на главной странице.

3. **Поддерживаемые методы входа**:
   - **Telegram**: Авторизация по ссылке входа, полученной от `@moyoprizvanie_bot` после отправки контакта (команда `/start` и кнопка «Поделиться контактом»), сканирования QR-кода или через встроенный браузерный виджет.
   - **Мессенджер МАКС (MAX)**: Авторизация через официального бота `@maxid_bot` в российском мессенджере МАКС (`max.ru`). Пользователь заходит в чат с ботом по ссылке/QR-коду, отправляет свой контакт с помощью специальной кнопки `request_contact` и получает персональную ссылку для быстрого входа в ЛК.
   - **Автоматическая регистрация (на лету)**: Если при отправке контакта боту Telegram или МАКС номер телефона пользователя не найден в базе данных, система автоматически регистрирует его:
     - Создается запись пользователя (роль `STUDENT`, привязывается соответствующий `telegramId` или `maxUserId`).
     - Создается связанный объект `Account` с провайдером `telegram` или `maxid`.
     - Генерируется сессионный токен Better Auth, создается сессия в БД со сроком действия 7 дней.
     - Отправляется ссылка быстрого входа `/api/auth/telegram/callback?token=<sessionToken>`, которая автоматически инициализирует сессию в браузере пользователя при переходе по ней.
   - **Электронная почта**: Полностью удалена из соображений безопасности и упрощения пользовательского опыта (без длинных анкет и паролей).
