# UI/UX Pro Max — Design Intelligence

Этот скилл регламентирует принципы проектирования премиального пользовательского опыта (UX), визуального дизайна (UI) и анимаций в проекте, а также содержит готовые решения для верстки и стилизации сложных интерактивных компонентов.

## 1. Цветовая палитра и Типографика

- **Отказ от дефолтных цветов:** Никаких стандартных ярких цветов Tailwind (`bg-yellow-500`, `bg-red-500`). Используйте благородные приглушенные оттенки.
- **Песочно-коричневый (Brand Accent):** Вместо стандартного желтого цвета используйте песочно-коричневый `#C4A484` для шагов коучинга, акцентов и сообщений наставника.
- **Контрастность в светлой теме:** При использовании полупрозрачных фонов (`bg-color/15`) в светлой теме цвет шрифта Tailwind (`text-color-400`) сливается с фоном. В `globals.css` обязательно прописывайте контрастные оверрайды на глубокие цвета (темно-зеленый `#15803d`, темно-синий `#1e3a8a`, глубокий кофейный `#3E2723`).
- **Шрифты:** В проекте используются три основных шрифта:
  - `Manrope` (основной текст, интерфейсы)
  - `Prata` (заголовки, премиальный стиль)
  - `Marck Script` (подписи, рукописные акценты)

## 2. Физика и Тайминги Анимаций (Framer Motion, GSAP)

- **Назначение движения:** Анимация должна быть функциональной (подсказывать иерархию, давать обратную связь, удерживать внимание), а не просто декоративной.
- **Временные рамки:** Оптимальная длительность UI-анимаций — **200–500 мс**. Менее 100 мс — незаметно, более 1000 мс — создает ощущение медлительности интерфейса.
- **Премиальный Easing (Физика):** Избегайте линейных переходов. Используйте премиальный cubic-bezier для плавного разгона и замедления:
  - В Framer Motion: `ease: [0.16, 1, 0.3, 1]` (плавный easeOut).
- **Ступенчатое появление (Stagger):** При рендеринге списков или сеток используйте `staggerChildren` (задержка около `0.08s - 0.12s` между элементами).
- **Живые интерфейсы:** Добавляйте анимированные фоновые блики (`motion.div` с медленным изменением градиента и масштаба) в личный кабинет и важные карточки.

## 3. Стилизация SVG и Canvas под Темную и Светлую Темы

- **Косело Призвания (RIASEC):** Чтобы избежать нечитаемости текста и слияния линий SVG-графики в светлой теме, **не используйте** жестко заданные hex-значения в параметрах `fill` или `stroke` внутри JSX/React-кода.
- **Решение:** 
  1. Задавайте адаптивные классы (например, `className="theme-wheel-label-active"`).
  2. Присваивайте системные CSS-переменные (например, `stroke="var(--border-subtle)"`).
  3. Прописывайте оверрайды `fill` и `stroke` в `globals.css` с использованием `!important` под селекторы `[data-theme="light"]` и `[data-theme="dark"]`.

## 4. Обход ограничений Tailwind CSS при смене тем

- В проектах с кастомными классами (например, `bg-white/[0.02]`, `border-white/5`), стандартные классы переопределения (такие как `.bg-white\/5` или `.border-white\/10`) не срабатывают автоматически.
- **Решение:** Явно экранируйте символы слеша и скобок в CSS-селекторах внутри глобальных стилей:
  ```css
  [data-theme="light"] .bg-white\/\[0\.02\] {
    background-color: rgba(0, 0, 0, 0.02) !important;
  }
  [data-theme="light"] .border-white\/5 {
    border-color: rgba(0, 0, 0, 0.05) !important;
  }
  ```

## 5. Адаптивность сеток и Видеофоны

- **Двухколоночная сетка для списков в боковых карточках:** Чтобы длинные списки элементов (например, 8 сфер Колеса Призвания) не выходили за рамки карточки и не обрезались доком/панелью задач на экранах ноутбуков, организуйте их в две колонки (`grid grid-cols-2`), что сокращает высоту списка вдвое и улучшает эргономику.
- **Адаптивный видеофон в мульти-темных интерфейсах:** При добавлении фонового видео для светлой и темной тем используйте два тега `<video>` с соответствующими классами видимости (`.theme-light-only` и `.theme-dark-only`) и регулируемой прозрачностью (`opacity`). Это позволяет мягко смешивать анимацию видео с градиентами подложки и сохранять высокую контрастность текста на переднем плане.
