# Скилл: База знаний по дизайну (Design Bible & RAG Engine)

Этот скилл служит системным справочником по визуальному дизайну, интерактивным сеткам, когнитивным законам UX, типографике и методам оценки макетов.

## БЛОК 1. КАРТА НАВЫКОВ ДИЗАЙНЕРА (SKILL CARDS)

### Skill Card 1.1: Visual Hierarchy (Визуальная иерархия)
* **ID:** DB-SKILL-001
* **Описание:** Управление вниманием пользователя на двухмерной плоскости экрана посредством распределения визуального веса элементов. Навык опирается на физиологию человеческого зрения (саккады, фовеальное и периферическое зрение) и законы гештальт-психологии.
* **Цель:** Провести взгляд пользователя по заранее спроектированному визуальному пути (Visual Path) для последовательного считывания информации и бесконфликтного достижения целевого действия (CTA).
* **Чек-лист оценки:**
  - [ ] Выделен один доминантный элемент на первом экране, занимающий максимальный визуальный вес.
  - [ ] Второстепенные элементы (субдоминанты) уступают доминанте по визуальному весу не менее чем на 30%.
  - [ ] Внутренние отступы между элементами блока меньше, чем внешние отступы между самими блоками (Закон близости).
  - [ ] Blur-тест: при размытии интерфейса на 20px траектория взгляда остается интуитивно понятной.

### Skill Card 1.2: Typography Design (Проектирование типографики)
* **ID:** DB-SKILL-002
* **Описание:** Навык выбора, масштабирования, верстки и технологической настройки шрифтовых семейств на цифровых носителях. Основан на понимании удобочитаемости (Legibility) и читаемости (Readability).
* **Чек-лист оценки:**
  - [ ] Шрифтовая пара подобрана с учетом характера бренда и проверена на читаемость в малых кеглях (9-12px).
  - [ ] Длина строки (Measure) для длинных текстов на десктопе укладывается в диапазон 45–75 символов (включая пробелы).
  - [ ] Интерлиньяж (Line height) для основного текста составляет от 1.4 до 1.6 от размера шрифта.
  - [ ] Использована фиксированная математическая шкала масштабирования (Type Scale: 1.250 / Major Third).

### Skill Card 1.3: Interaction Design (Интеракционный дизайн)
* **ID:** DB-SKILL-003
* **Описание:** Проектирование поведения системы в ответ на действия пользователя. Базируется на теории обратной связи, ментальных моделях и проектировании микросостояний компонентов.
* **Чек-лист оценки:**
  - [ ] Любое интерактивное действие имеет мгновенный визуальный отклик (Hover, Active, Focus, Disabled, Loading states).
  - [ ] Время отклика анимации (Duration) не превышает 300мс на десктопе и 200мс на мобильных устройствах.
  - [ ] Интерактивные паттерны соответствуют общепринятым стандартам (например, нажатие на логотип возвращает на главную).

### Skill Card 1.4: Behavioral Design (Поведенческий дизайн)
* **ID:** DB-SKILL-004
* **Описание:** Проектирование интерфейсных решений на основе когнитивной психологии и поведенческой экономики для мягкого изменения привычек и выбора пользователей (Nudging).
* **Чек-лист оценки:**
  - [ ] Интерфейс предоставляет пользователю чувство автономии и контроля над действиями.
  - [ ] Триггеры к действию возникают в момент наивысшей мотивации и минимального когнитивного барьера (модель поведения Фогга $B=MAP$).
  - [ ] Отсутствуют скрытые платежи, автоматически добавляемые товары и затрудненные пути отмены подписки (Dark Patterns).

### Skill Card 1.5: Accessibility Implementation (Обеспечение доступности)
* **ID:** DB-SKILL-005
* **Описание:** Проектирование и верстка цифровых продуктов с учетом потребностей людей с ограниченными возможностями (зрительными, слуховыми, моторными, когнитивными). Ориентировано на стандарты WCAG 2.1/2.2.
* **Чек-лист оценки:**
  - [ ] Контрастность текста по отношению к фону соответствует стандарту WCAG AA (минимум 4.5:1 для обычного текста, 3.1 для крупного).
  - [ ] Все интерактивные элементы имеют четко выраженное состояние фокуса при навигации клавишей Tab.
  - [ ] Изображения снабжены содержательными альтернативными описаниями (alt атрибуты).
  - [ ] Логическая структура документа размечена семантическими тегами (H1-H6, `<main>`, `<nav>`, `<aside>`).

### Skill Card 1.6: Conversion Design (Конверсионный дизайн)
* **ID:** DB-SKILL-006
* **Описание:** Проектирование целевых страниц и интерфейсов с целью максимизации соотношения совершивших целевое действие пользователей к общему числу посетителей. Базируется на снижении трения (Friction) и повышении ценности.
* **Чек-лист оценки:**
  - [ ] На первом экране четко сформулировано уникальное ценностное предложение (UVP).
  - [ ] Основная кнопка CTA (Call to Action) является самым контрастным элементом страницы и находится в зоне прямой видимости.
  - [ ] В форме сбора данных удалены все необязательные поля.

## БЛОК 2. КОГНИТИВНАЯ ПСИХОЛОГИЯ (LAWS OF UX)

* **Закон Фиттса (Fitts's Law):** Время достижения цели зависит от расстояния до нее и ее размера:
  $$MT = a + b \log_2 \left( \frac{2D}{W} \right)$$
  *Применение:* Тач-зоны кнопок на мобильных устройствах должны быть $\ge 48\times48\text{px}$. Кнопки с высоким приоритетом расположены ближе к естественной траектории движения пальца.

* **Закон Хика (Hick's Law):** Время принятия решения увеличивается логарифмически с ростом числа альтернатив:
  $$T = b \cdot \log_2(n + 1)$$
  *Применение:* Снижение когнитивной нагрузки. Ограничение выбора (не более 3-4 тарифов, пошаговые формы вместо длинных списков полей).

* **Закон Миллера (Miller's Law):** Человек может удерживать в кратковременной памяти $7 \pm 2$ элементов одновременно.
  *Применение:* Группировка длинных цифр по блокам (чанчинг). Не более 7 пунктов в основном меню.

* **Закон Якоба (Jakob's Law):** Пользователи предпочитают, чтобы ваш сайт работал точно так же, как и все остальные сайты, которые они уже знают (например, корзина справа сверху, логотип слева сверху возвращает на главную).

* **Закон Peak-End Rule (Закон пика и конца):** Люди оценивают свой опыт в основном по двум точкам: на пике (самый эмоциональный момент) и в самом конце взаимодействия (например, страница успеха с анимацией конфетти при успешном завершении теста).

* **Эффект изоляции (Von Restorff Effect):** Из нескольких сходных объектов лучше всего запоминается тот, который выделяется на фоне остальных своим цветом, формой или размером.

* **Эффект привязки (Anchoring):** Склонность человека полагаться на первую предложенную информацию при принятии решений.

## БЛОК 4. МАТРИЦА ОЦЕНКИ КАЧЕСТВА (Design Critique Heuristics)

Каждый макет оценивается по 5-балльной шкале по следующим критериям:

| Критерий | Вес | Формула / Условие | Критический показатель |
| :--- | :--- | :--- | :--- |
| **Contrast (WCAG AA)** | 0.25 | $C = (L_1 + 0.05) / (L_2 + 0.05)$ | $\ge 4.5:1$ для обычного текста |
| **Typographic Rhythm** | 0.20 | $H_{line} = LineHeight \times FontSize$ | Должен быть кратен 4px или 8px |
| **Focal Points** | 0.20 | Blur Test 20px | $\le 2$ фокусные точки на экране |
| **Interaction Target** | 0.20 | Размер кликабельной области | $\ge 48\text{px}$ на мобильных |
| **Cognitive Load** | 0.15 | Option Selection Count (Hick's Law) | Меньше 5 вариантов у CTA |
