# Systematic Debugging — Систематическая отладка (Defense in Depth)

Этот навык предназначен для пошагового анализа, изоляции и устранения системных ошибок в проекте. Вместо случайных правок («vibe debugging») внедряется строгий научный метод диагностики и многослойного контроля.

## 1. 4-этапный алгоритм диагностики (Observe-Analyze-Fix-Verify)

При возникновении ошибки или неожиданного поведения системы всегда следуйте следующей последовательности:

1. **Сбор фактов (Observe):**
   - Получите полный стек вызовов (Stack Trace) и лог ошибки.
   - Изучите логи сервера из `logger` или логи браузера.
   - Выявите точный эндпоинт, компонент или функцию, где происходит сбой.
   
2. **Анализ первопричин (Analyze):**
   - Постройте гипотезу: почему это происходит? (Ошибка типов, null/undefined, проблемы с базой данных, состояние сессии Better Auth, гонка процессов).
   - Не делайте домыслов. Проверьте гипотезу, прочитав соответствующий файл или выполнив точечные тесты.
   
3. **Локализованное исправление (Fix):**
   - Внесите минимальные, точные изменения для исправления бага.
   - Добавьте защитное программирование (guards, fallback-значения).
   
4. **Многоуровневая верификация (Verify):**
   - Проверьте сборку проекта (`npm run build` или `ruff check .` для Python).
   - Убедитесь, что баг не воспроизводится.

## 2. Подход «Защита в глубину» (Defense in Depth)

Никогда не полагайтесь на валидацию только в одной точке системы. Ошибки должны предотвращаться на нескольких уровнях:

```
[Входные данные] ──► 1. Валидация на входе (Zod / API контракты)
                     └──► 2. Бизнес-логика (Типы TypeScript, Guard clauses)
                          └──► 3. Защита БД (Призма/Supabase, Constraints, Темы)
                               └──► 4. Логирование и алертинг (logger / Incident log)
```

- **Уровень 1 (Entry):** Проверяйте типы параметров API-запросов (`zod`, приведение типов, фильтрация стоп-слов в коуч-чате).
- **Уровень 2 (Logic):** Проверяйте существование сущностей перед вызовом их методов (например, `typeof val === 'string'` для JSON-полей Prisma).
- **Level 3 (Database):** Используйте транзакции для связанных записей и правильные внешние ключи.
- **Level 4 (Monitoring):** Логируйте подробный контекст ошибки (свойства, ID сессии) с помощью `logger.error()`, скрывая чувствительные данные (пароли, личные токены).
